# v0.2 — Duplicación de UTMs + Nombres de plantilla + Tooltips

## Cambios

### 1. Nombre de plantilla (`template_name`)
- Nuevo campo opcional en el formulario "Crear UTM", encima de los campos principales.
- Permite ponerle un nombre reutilizable a una UTM: `the-idea`, `the-story`, `newsletter-semanal`, etc.
- Se guarda en la base de datos como `template_name`.
- En el historial, los UTMs con nombre de plantilla aparecen destacados: `🏷️ the-idea — google / cpc`.

**Migración requerida en Supabase:**
```sql
ALTER TABLE utms ADD COLUMN template_name TEXT;
```

### 2. Duplicación de UTMs desde el historial
- Cada UTM en el historial tiene ahora dos botones: `📋 Duplicar` y `🗑️ Eliminar`.
- Al hacer clic en "Duplicar" se despliega un formulario inline dentro del mismo expander, pre-llenado con todos los campos del UTM original.
- El usuario edita solo los campos que necesita cambiar (URL, source, medium, nombre, etc.).
- Al guardar, se crea una entrada nueva en el historial sin modificar el original.
- El botón "Cancelar" cierra el formulario sin guardar.

### 3. Tooltips descriptivos en el formulario
- Todos los campos del formulario "Crear UTM" tienen ahora un tooltip (`?`) con descripción y ejemplos reales:
  - **Website URL** — URL de destino con http/https.
  - **Campaign Source** — plataforma o remitente (`google`, `linkedin`, `newsletter`).
  - **Campaign Medium** — canal de tráfico (`cpc`, `email`, `social`, `banner`).
  - **Campaign Name** — nombre de campaña con convención de guiones bajos.
  - **Campaign ID** — identificador para vincular con plataformas de pauta.
  - **Campaign Term** — palabra clave para campañas SEM.
  - **Campaign Content** — diferencia variantes para pruebas A/B.
  - **Descripción** — contexto interno, no aparece en la URL.

## Archivos modificados
- `app.py` — formulario, historial, session state
- `core/database.py` — `save_utm()` acepta el nuevo parámetro `template_name`
